Operations Guide

Runbook

Start, validate, and demo AHMS in a reliable order.

Recommended Startup Order

  1. Java backend
  2. Python analysis service
  3. C++ simulator
  4. C# control panel
  5. Website preview

Java Backend

Working directory: java-backend

mvn spring-boot:run

Checks:
GET /api/health
GET /api/sensors/latest
GET /api/alerts

Python Analysis

Working directory: python-analysis

pip install -r requirements.txt
python analyzer.py

Checks:
GET /health
POST /api/analysis/run
GET /api/analysis/report

C++ Simulator

Working directory: cpp-simulator

cmake -S . -B build
cmake --build build
.\build\sensor-simulator.exe

C# Control Panel

Working directory: csharp-control-panel

dotnet run

Troubleshooting

  • If analysis fails, confirm backend is running on port 8080.
  • If latest values are empty, test GET /api/sensors/latest.
  • If simulator cannot send, confirm localhost:8080 is reachable.
  • If alerts do not appear, call POST /api/analysis/run manually.
