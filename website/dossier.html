System Summary

  • Ingestion: Java Spring Boot
  • Analysis: Python Flask
  • Control: C# WinForms
  • Transport Source: C++
  • Integrity: Assembly
  • Logic Model: VHDL

Subsystem Ownership

Operations Sequence

[01] Start Java backend on port 8080
[02] Start Python analysis service on port 8000
[03] Run C++ simulator and stream packets
[04] Trigger analysis or request report
[05] Launch C# control panel and observe status
[06] Optionally simulate VHDL alert circuit

Operational Value

  • The backend now exposes a health endpoint with record and alert counts.
  • The Python service reports backend reachability inside its own health response.
  • The analysis service creates the report directory safely before writing output.
  • The repo includes a runbook and component guides for demo and handoff.
