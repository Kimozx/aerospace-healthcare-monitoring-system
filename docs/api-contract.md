# API Contract

This document defines the payloads and endpoints used across the monitoring system.

## Sensor Packet

The simulator sends the following JSON payload to the backend:

```json
{
  "temperature": 84.7,
  "pressure": 103.4,
  "vibration": 73.2,
  "oxygenLevel": 88.9,
  "heartRate": 112,
  "batteryLevel": 18.4,
  "timestamp": "2026-06-10T15:45:10Z",
  "checksum": 201
}
```

## Backend Endpoints

### `POST /api/sensors`

Receives telemetry from the C++ simulator.

Expected body:

```json
{
  "temperature": 72.4,
  "pressure": 101.2,
  "vibration": 22.5,
  "oxygenLevel": 97.0,
  "heartRate": 76,
  "batteryLevel": 88.0,
  "timestamp": "2026-06-10T14:30:05Z",
  "checksum": 173
}
```

### `GET /api/sensors`

Returns all stored sensor records.

### `GET /api/sensors/latest`

Returns the newest sensor record.

### `POST /api/alerts`

Stores an alert generated by the Python analysis service.

Expected body:

```json
{
  "alertType": "LOW_OXYGEN",
  "message": "Oxygen dropped to 88.9",
  "severity": "CRITICAL",
  "sensorTimestamp": "2026-06-10T15:45:10Z"
}
```

### `GET /api/alerts`

Returns the alert history.

## Python Analysis Endpoints

### `GET /health`

Returns service health status.

### `POST /api/analysis/run`

Triggers analysis, generates a report, and posts newly detected alerts to the backend.

### `GET /api/analysis/report`

Runs analysis and returns the latest plain-text report.

## Checksum Contract

The checksum is calculated over the canonical payload string below:

```text
timestamp|temperature|pressure|vibration|oxygenLevel|heartRate|batteryLevel
```

The numeric values are formatted to two decimal places before summing bytes.

Example canonical payload:

```text
2026-06-10T15:45:10Z|84.70|103.40|73.20|88.90|112|18.40
```

The Assembly implementation computes the byte-sum checksum in the simulator, and the Java backend recomputes the same checksum before accepting the packet.
